📦 Guia Completo: Atualizar Versão, Push e Build pelo GitHub Actions
1. 📝 Atualizar a Versão no Package.json
powershell
# Navegar até a raiz do projeto
cd "D:\Download\Cora Cowork UI Desktop\Cora Cowork UI Deskto 1.0"

# Abrir o package.json para editar
notepad package.json
Altere a versão no arquivo package.json:

json
{
  "name": "CoraCowork",
  "version": "0.1.4",  // ← Mude para a nova versão
  "coracoreVersion": "v0.2.5"
}
2. 📝 Atualizar o electron-builder.yml
powershell
# Abrir o electron-builder.yml
notepad packages/desktop/electron-builder.yml
Procure e atualize a versão se houver referência:

yaml
# Se houver, atualize para a mesma versão
version: 0.1.4
3. 📝 Atualizar o CHANGELOG.md (opcional)
powershell
# Abrir o CHANGELOG
notepad CHANGELOG.md
Adicione as mudanças da nova versão:

markdown
## [0.1.4] - 2026-07-16

### Added
- Suporte a Windows ARM64 no CI
- Build para Linux ARM64

### Fixed
- Corrigido asset naming do CoraCore
- Corrigido binary resolver para coracore.exe
4. 📝 Verificar as mudanças
powershell
# Ver o que foi modificado
git status

# Ver as mudanças específicas
git diff package.json
git diff packages/desktop/electron-builder.yml
5. 📦 Commitar as mudanças
powershell
# Adicionar arquivos modificados
git add package.json
git add packages/desktop/electron-builder.yml
git add CHANGELOG.md

# Ou adicionar tudo
git add .

# Commitar com mensagem descritiva
git commit -m "chore: bump version to 0.1.4"
6. 🏷️ Criar a Tag de Release
powershell
# Criar tag com a nova versão
git tag -a v0.1.4 -m "Release v0.1.4 - Suporte a ARM64"

# Verificar a tag criada
git tag -l
7. 🚀 Push para o GitHub
powershell
# Push do código e das tags
git push origin main
git push origin v0.1.4

# Ou push com tudo junto
git push --follow-tags
8. 📊 Verificar o Workflow no GitHub
powershell
# Abrir no navegador
Start-Process "https://github.com/coracowork/cora-cowork-desktop/actions"
Acesse também: https://github.com/coracowork/cora-cowork-desktop/actions
